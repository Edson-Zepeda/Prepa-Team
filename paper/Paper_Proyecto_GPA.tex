\begin{abstract}
Este artículo presenta un sistema de inteligencia artificial, basado en aprendizaje automático, para predecir y analizar el rendimiento académico de estudiantes y transformar la salida del modelo en recomendaciones accionables de intervención temprana. El estudio utiliza un conjunto de datos tabular de 2,392 estudiantes y 15 variables, con información de edad, tiempo semanal de estudio, ausencias, tutorías, apoyo parental, actividades extracurriculares y GPA. El enfoque combina dos tareas complementarias: una tarea de regresión para estimar GPA y una tarea de clasificación calibrada para estimar la probabilidad de alcanzar buen rendimiento, definido como GPA $\geq 2.5$. Se compararon LinearRegression, Random Forest, SVR y XGBoost. El mejor modelo de regresión fue LinearRegression, con RMSE = 0.1963 y $R^2$ = 0.9534 en prueba. Tras afinar XGBoost, su RMSE de prueba se redujo a 0.2028 y sus predicciones se mantuvieron muy cercanas a las del modelo lineal, con correlación de 0.9979, pero aún así no lo superó en generalización. La prueba de ablación sin la variable Absences incrementó el RMSE del modelo lineal de 0.1963 a 0.8692, confirmando que las ausencias son el factor dominante. Finalmente, se implementó un motor de recomendaciones que simula intervenciones, excluye variables sensibles o no accionables y prioriza acciones con mayor impacto estimado en GPA y probabilidad de buen rendimiento. Los resultados sugieren que el valor práctico del sistema no reside solo en predecir, sino en convertir la predicción en apoyo supervisado para estudiantes y tutores.
\end{abstract}

\section{Introducción}

La predicción temprana del rendimiento académico es una aplicación relevante de Educational Data Mining porque permite identificar estudiantes con riesgo y orientar recursos de apoyo antes de que el bajo desempeño se consolide. La literatura reciente muestra que los modelos de aprendizaje automático pueden ser útiles para anticipar resultados académicos y apoyar decisiones educativas \cite{khairy2024prediction}. Sin embargo, un sistema que solo entrega una etiqueta o un puntaje tiene utilidad limitada: para una escuela, un tutor o un estudiante, la pregunta central no es solamente ``cuál será el GPA'', sino ``qué se puede mejorar para elevar la probabilidad de buen rendimiento''.

Este trabajo responde a ese problema mediante un flujo reproducible que combina predicción, interpretación y recomendación. Primero, se entrena un conjunto de modelos de regresión para estimar el GPA. Segundo, se analiza por qué XGBoost no supera al modelo lineal aunque sus predicciones se comporten de manera muy parecida. Tercero, se realiza una prueba de ablación quitando Absences para medir la dependencia del modelo respecto a esa variable. Cuarto, se entrena un clasificador calibrado para estimar la probabilidad de alcanzar buen rendimiento, definido como GPA $\geq 2.5$. Quinto, se construye un motor de recomendaciones que prueba escenarios de intervención sobre variables accionables.

La contribución principal del artículo es proponer una ruta práctica para convertir un modelo predictivo de desempeño académico en una herramienta de apoyo. La propuesta evita recomendar sobre variables sensibles o no modificables, como Gender, Ethnicity, StudentID y GradeClass, y se enfoca en variables donde una intervención educativa podría tener sentido: ausencias, horas de estudio, tutorías, apoyo parental y participación en actividades. Esta distinción es importante porque los modelos educativos pueden generar riesgos de sesgo o uso indebido si se convierten en decisiones automáticas sin supervisión humana \cite{hardt2016equality,barocas2023fairness}.

\section{Trabajo relacionado}

El uso de modelos de clasificacion y regresion para estimar desempeno estudiantil es un tema consolidado en Educational Data Mining. Khairy et al. \cite{khairy2024prediction} muestran que algoritmos de mineria de datos pueden apoyar la prediccion de resultados de examenes. Otros trabajos comparan modelos como Decision Tree, Random Forest, KNN, Naive Bayes y SVM para rendimiento academico, con el objetivo de apoyar sistemas de alerta temprana y programas de intervencion. En paralelo, metodos como Random Forest \cite{breiman2001random}, Support Vector Machines \cite{cortes1995support} y XGBoost \cite{chen2016xgboost} se han utilizado ampliamente por su capacidad de modelar relaciones no lineales y capturar interacciones.

El vacio que atiende este proyecto no es solo obtener una metrica alta, sino conectar el resultado predictivo con acciones interpretables. Un modelo con buen desempeno puede seguir siendo poco util si no traduce su salida en recomendaciones comprensibles y accionables. Por ello, este trabajo incorpora un modulo contrafactual simple: modifica variables accionables, recalcula el GPA estimado y recalcula la probabilidad de buen rendimiento para priorizar planes de mejora.

\section{Datos}

Se utilizo el archivo local \texttt{student\_performance.csv}. La fuente del dataset es Kaggle, ``Students Performance Dataset'', de Rabie El Kharoua \cite{kharoua2024students}. El archivo local coincide con el esquema esperado: 2,392 registros y 15 columnas.

Las columnas disponibles son StudentID, Age, Gender, Ethnicity, ParentalEducation, StudyTimeWeekly, Absences, Tutoring, ParentalSupport, Extracurricular, Sports, Music, Volunteering, GPA y GradeClass. No se observaron valores faltantes. El GPA observado en el archivo local se encuentra en el rango empirico de 0.0 a 4.0, con media 1.9062. Para construir el objetivo de clasificacion se definio buen rendimiento como GPA $\geq 2.5$, con 706 estudiantes en la clase positiva y 1,686 por debajo del umbral.

\section{Metodologia}

\subsection{Preparacion de datos}

Para la tarea de regresion se uso GPA como variable objetivo y se eliminaron StudentID y GradeClass. StudentID es un identificador sin valor predictivo generalizable, mientras que GradeClass codifica directamente el rendimiento, por lo que incluirla al predecir GPA generaria fuga de informacion. Se conservaron variables demograficas codificadas para el analisis predictivo general, pero se excluyeron del motor de recomendaciones.

El preprocesamiento se implemento con pipelines de scikit-learn \cite{pedregosa2011scikit}. Las variables numericas se imputaron con mediana y se escalaron con StandardScaler. Las variables categoricas codificadas se imputaron con la moda y se transformaron mediante OneHotEncoder. El conjunto se dividio en entrenamiento y prueba con particion 80/20 y \texttt{random\_state = 42}. La estabilidad se evaluo mediante validacion cruzada de 5 particiones.

\subsection{Modelos de regresion}

Se entrenaron cuatro modelos para estimar GPA: LinearRegression, RandomForestRegressor, SVR con kernel RBF y XGBRegressor. Las metricas principales fueron MAE, RMSE y $R^2$. Se selecciono RMSE como metrica de comparacion principal porque penaliza errores grandes en GPA, lo cual es relevante en escenarios de alerta temprana.

\subsection{Analisis de XGBoost y ablacion}

El comportamiento de XGBoost se analizo de forma especifica porque sus resultados eran cercanos al modelo lineal, pero no superiores. Para ello se compararon metricas de entrenamiento, prueba y validacion cruzada. Ademas, se midio la correlacion entre las predicciones de LinearRegression y XGBoost. Para estudiar la dependencia del modelo respecto a las ausencias, se repitio el entrenamiento eliminando Absences y se comparo el RMSE resultante contra la linea base.

\subsection{Clasificador de buen rendimiento}

Para producir mensajes del tipo ``aumenta tu probabilidad de buen rendimiento'', se entreno una segunda tarea de clasificacion. La variable objetivo se definio como:

\begin{equation}
y_{good} =
\begin{cases}
1, & \text{si } GPA \geq 2.5\\
0, & \text{si } GPA < 2.5
\end{cases}
\end{equation}

Se evaluaron LogisticRegression, RandomForestClassifier, HistGradientBoostingClassifier y XGBClassifier. El mejor clasificador se selecciono por ROC AUC en validacion cruzada y posteriormente se calibro con CalibratedClassifierCV usando calibracion sigmoidal. La calibracion se incluyo porque el motor de recomendaciones usa probabilidades para comparar escenarios, no solo clases binarias.

\subsection{Motor de recomendaciones}

El motor de recomendaciones usa solo variables accionables o de contexto no sensible: Age, StudyTimeWeekly, Absences, ParentalEducation, Tutoring, ParentalSupport, Extracurricular, Sports, Music y Volunteering. Se excluyen StudentID, GradeClass, Gender y Ethnicity. En el producto, ParentalEducation se conserva solo como contexto y no se recomienda de forma directa; ParentalSupport se interpreta como una escala ordinal cualitativa de apoyo familiar, desde muy bajo hasta muy alto. Las intervenciones simuladas incluyen reducir ausencias, aumentar horas de estudio, activar tutorias, reforzar apoyo familiar y activar una actividad complementaria. Para escenarios intensivos, el motor tambien considera reducir ausencias a un maximo de 5 y aumentar el estudio hasta 20 horas semanales.

Para cada plan se calcula GPA actual estimado, GPA posterior estimado, probabilidad actual de buen rendimiento, probabilidad posterior y diferencia esperada. El sistema ordena planes por incremento de probabilidad, incremento de GPA y esfuerzo estimado. Las salidas del regresor de apoyo se acotan al rango valido de GPA [0, 4] para evitar mensajes imposibles en casos extremos.

\section{Discusion}

Los resultados muestran que el modelo mas complejo no siempre es el mas adecuado. XGBoost tiene mayor capacidad expresiva y ajusta mejor el entrenamiento, pero el modelo lineal generaliza mejor en prueba y validacion cruzada. Esto es coherente con la estructura del dataset: la relacion dominante entre ausencias y GPA parece suficientemente fuerte y casi lineal para que un modelo simple capture gran parte de la senal.

La prueba sin Absences es el hallazgo mas importante para la interpretacion educativa. Si se elimina esa variable, el desempeno cae drasticamente. Esto tiene dos implicaciones. Primero, el modelo depende de una variable altamente informativa, por lo que debe evitarse presentar el sistema como una inteligencia general del rendimiento escolar. Segundo, la reduccion de ausencias es una intervencion prioritaria para estudiantes en riesgo, aunque el modelo por si solo no prueba causalidad.

El componente de recomendacion mejora la utilidad del proyecto porque transforma resultados predictivos en escenarios de accion. En vez de limitarse a mostrar un GPA estimado, el sistema compara planes y cuantifica el cambio esperado. Esta capa convierte el analisis en una herramienta potencial para tutores, coordinadores o estudiantes, siempre que se use como apoyo supervisado.

\section{Consideraciones eticas y limitaciones}

El sistema no debe usarse para sancionar, excluir o etiquetar automaticamente estudiantes. Sus salidas son simulaciones del modelo, no decisiones academicas definitivas. Ademas, aunque se excluyen Gender y Ethnicity del motor de recomendaciones, el conjunto de datos puede contener sesgos historicos o relaciones indirectas entre variables. Por ello, el uso institucional requeriria auditoria periodica, revision humana y validacion con datos locales.

Existen varias limitaciones. Primero, aunque la fuente del dataset fue confirmada como Kaggle \cite{kharoua2024students}, antes de uso institucional se requiere validacion con datos escolares locales. Segundo, el conjunto de datos es transversal y no longitudinal; por tanto, el modelo no estima evolucion temporal. Tercero, las recomendaciones son contrafactuales simples, no evidencia causal. Cuarto, faltan variables relevantes como salud, carga academica, condiciones socioeconomicas, calidad docente y contexto institucional. Quinto, el caso de demostracion intensivo maximiza rendimiento bajo reglas simuladas, pero su factibilidad real dependera del estudiante, tutor y escuela.

\section{Conclusiones}

Este articulo presento un sistema predictivo e interpretable para estimar GPA y generar recomendaciones academicas accionables. El mejor regresor fue LinearRegression, con RMSE = 0.1963 y $R^2$ = 0.9534 en prueba. XGBoost mejoro tras afinacion hasta RMSE = 0.2028, pero mantuvo una brecha de generalizacion mayor y no se selecciono como modelo final. La ablacion sin Absences confirmo que las ausencias son el factor mas influyente. El clasificador calibrado logro ROC AUC = 0.9871 y permitio estimar probabilidades de buen rendimiento para comparar escenarios.

La contribucion practica del proyecto es conectar prediccion y accion: el sistema identifica factores de riesgo, simula planes de mejora y genera mensajes comprensibles para orientar intervenciones. El siguiente paso es convertir el notebook en una plataforma escolar demo con carga de CSV, ficha individual de estudiante, simulador de intervenciones, reporte para tutores y monitoreo etico de subgrupos.
